# Security Policy

## Supported Versions

As DAM is currently in early active development, security updates are provided only for the latest released version of the CLI.

Please ensure you are running the most recent version before reporting a vulnerability.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

We take the security of DAM and its integrated credential management systems seriously. If you discover a potential security vulnerability, **please do not report it publicly through GitHub Issues or Discussions**.

Instead, report the vulnerability privately by email so we have an opportunity to investigate and address the issue before any public disclosure.

### How to Report

Please send an email to:

`contacts.oneam@gmail.com`

Use the following **exact subject line**:

`DAM-SECURITY-VULNERABILITY-REPORT`

In the body of your report, please include the following information:

* **Description of the vulnerability** — Clearly explain the issue and, if possible, how it can be exploited.
* **DAM version** — Include the output of `dam --version`.
* **Operating system** — Specify the OS and version you are using.
* **Steps to reproduce** — Provide clear, step-by-step instructions that allow us to reproduce the issue.
* **Potential impact** — Describe the security impact and any exploit scenarios you have identified.

### Response Timeline

You should expect a preliminary acknowledgment within **3-5 business days** of submitting your report.

We will keep you informed as we investigate the vulnerability, assess its impact, and work toward developing and releasing a fix.

Thank you for helping us keep DAM and its users secure.
